DockerHub Image CF Worker 加速

前期准备

  1. 购买一个域名，如：turing.com
  2. 在CF中添加站点，完成DNS的解析

代码配置

  • CF-Workers-docker.io，一个基于 Cloudflare Workers 的 Docker 镜像代理工具
    • 优化版代码
  • 复制_worker.js代码：
  • 这段脚本实现了一个反向代理服务器，用于在客户端和 Docker Hub 之间转发请求和响应，同时处理一些特定的头信息和重定向
  • 打开 CF 部署Workers

  • 创建完成后，编辑js代码

  • 将上述jS代码复制上去，修改let workers_url = 'https://turing.com'为自己的域名
  • 再进行访问，如果成功部署，页面会跳转到docker-hub
  • 再进行设置-触发器-添加路由
    • 路由支持通配符，一般设置为Workers项目名.域名
      • docker-hub.turing.com
    • 区域则是站点，添加的域名turing.com

  • 通过配置的路由，再次访问，如果成功部署，页面会跳转到docker-hub

拉取示例

  • 拉取仓库：docker pull docker-hub.turing.com/library/alpine:latest
  • 拉取成功后，docker images，查看
  • 简化拉取命令，配置docker镜像后，直接docker pull alpine:latest即可
